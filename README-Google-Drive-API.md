### 🔑 Підготовка: Отримання доступу до Google Drive API  
Оскільки додаток працюватиме автономно від твого імені, нам потрібно створити проект у консолі розробника Google та завантажити файл із доступами (Credentials).  
Ось покроковий план, що потрібно зробити зараз у браузері:  
- Перейди до [Google Cloud Console](https://console.cloud.google.com/).  
- Створи новий проект (наприклад, Personal-Kanban-Desktop).  
- У меню ліворуч перейди в розділ API & Services ➡️ Library, знайди в пошуку Google Drive API та натисни Enable (Увімкнути).  
- Перейди в розділ OAuth consent screen (Екран згоди):  
  - Обери тип External.
  - Заповни обов'язкові поля (назва додатка, твій email).
  - На кроці Test users обов'язково додай свій особистий Google-email (саме з цим диском додаток і буде синхронізуватися).  
- Перейди в розділ Credentials (Облікові дані):
  - Натисни Create Credentials ➡️ OAuth client ID.
  - Application type: Обери Desktop app.
  - Натисни Create та завантаж отриманий JSON-файл на свій комп'ютер.
  
### 📦 Як ми підключимо цей JSON-файл у проект?  
Коли ти завантажиш цей файл конфігурації від Google, найкраща практика — покласти його в корінь папки backend/ під назвою, наприклад, credentials.json (і ми обов'язково перевіримо, щоб він був заігнорений у Git). Наш бекенд через офіційну бібліотеку google-auth-oauthlib зчитає його і при першому запуску відкриє вікно браузера для одноразового підтвердження доступу.
